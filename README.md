# Doctor Handwritten Prescription Recognition using Classic ML

## 📖 Overview
This repository contains the implementation of an Optical Character Recognition (OCR) system specifically designed to decode doctors' handwritten prescriptions. To ensure a lightweight and highly interpretable system, this project avoids resource-heavy Deep Learning methods. Instead, it utilizes **Classical Machine Learning** algorithms combined with handcrafted **Feature Extraction** techniques.

This project is part of the **Research Methodology** coursework at Bina Nusantara University.

## 📊 Dataset
The dataset used in this project is the **[Doctors' Handwritten Prescription BD Dataset](https://www.kaggle.com/datasets/mamun1113/doctors-handwritten-prescription-bd-dataset)** from Kaggle. It contains handwritten drug names with 78 different classes (medicine names). 

*Note: Due to file size limitations, the dataset is not included in this repository. Please download it from Kaggle and place it in the `dataset/raw/` directory.*

## ⚙️ Methodology
Our system pipeline consists of the following stages:
1. **Preprocessing:** Grayscale conversion, Otsu's Binarization, Resizing to 64x64 pixels, and Normalization.
2. **Feature Extraction:** 
   * **HOG** (Histogram of Oriented Gradients) for structural patterns.
   * **LBP** (Local Binary Pattern) for texture strokes.
   * **Gabor Filters** for spatial frequency and orientation.
3. **Feature Fusion:** Concatenating HOG, LBP, and Gabor features into a single high-dimensional vector, followed by `StandardScaler`.
4. **Classification:** Tuning and evaluating models using Support Vector Machine (SVM), K-Nearest Neighbor (KNN), and Random Forest (RF) using `GridSearchCV` (5-Fold Cross-Validation).
